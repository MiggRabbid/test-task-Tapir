#### Задача

Разработать: Nuxt-приложение на основе предоставленного [макета](https://www.figma.com/design/DiLKO8e4HzMcpVlo51z6HS/%D0%A2%D0%B5%D1%81%D1%82%D0%BE%D0%B2%D0%BE%D0%B5-%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%B8%D0%B5?node-id=1-323&t=7qaDtGAzKRKlqk4V-0)

##### Технологии

Nuxt 4 + TypeScript

Эндпоинт для получения товаров: test-task-api.tapir.ws/products
Примеры использования:

- test-task-api.tapir.ws/products?page=1
- test-task-api.tapir.ws/products?page=2&limit=10

##### Функциональные требования

Шапка:

- Все ссылки — заглушки (href="#")
- «Бургер кнопка» никаких действий не производит. Мобильное меню реализовывать не требуется.
Главная страница:
- При клике на кнопку «В каталог» происходит переход на страницу каталога.

Каталог товаров:

- Вся карточка товара является ссылками с заглушкой (href="#").
- Кнопка «Добавить в избранное» в карточке товара визуально присутствует, но не выполняет никаких действий.
- Цена «x2» в карточке товара рассчитывается как <цена товара> / 2, округлить до целого в большую сторону
  - При первой загрузке страницы отображается первая страница товаров (SSR-рендеринг обязателен).
  - При нажатии кнопки «Показать ещё»:
    - кнопка скрывается;
    - вместо неё отображается индикатор «Загрузка...».
  - В случае успешного ответа:
    - Индикатор «Загрузка...» скрывается.
    - Новые товары добавляются к уже отображаемому списку
    - Если доступна следующая страница, повторно отображается кнопка «Показать ещё».
  - В случае сетевой ошибки:
    - Индикатор «Загрузка...» скрывается.
    - Под списком товаров отображается блок с текстом: «Произошла ошибка, попробуйте позже» и кнопка «Повторить», при нажатии которой выполняется повторная попытка получить данные.

Подвал:

- Все ссылки — заглушки (href="#").

##### Технические требования

- Поддержка SSR.
- Использование TypeScript.
- Адаптивная верстка в соответствии с макетом.
- Именование CSS-классов по методологии БЭМ (блок__элемент--модификатор).

#### Реализация

1. Реализованы две страницы приложения: главная (`/`) и каталог (`/catalog`) на файловом роутинге Nuxt.
2. Собраны адаптивные компоненты интерфейса: `header`, `footer`, карточки каталога и контролы постраничной догрузки.
3. Первая страница каталога загружается через SSR, последующие — по кнопке «Показать ещё» с состояниями `loading`, `success`, `error` и повторной попыткой.
4. На сервере реализован прокси-эндпоинт `server/api/products.get.ts` с валидацией query-параметров, ограничением `limit` и обработкой ошибок внешнего API.
5. Кнопка «Избранное» в карточке оставлена только как визуальный элемент (без изменения состояния), в соответствии с ТЗ.
6. Стили организованы на SCSS с дизайн-токенами и брейкпоинтами, классы оформлены по БЭМ.

#### Используемый стек

- Core:

  - Vue 3

- Styling:
  - SCSS (Sass Embedded)
- Server:

  - Makefile для shortcut-команд разработки

#### Структура проекта

```text
.
├── app                               // Исходный код Nuxt-приложения
│   ├── app.vue                       // Корневой компонент приложения
│   ├── assets                        // Шрифты и глобальные стили
│   │   ├── fonts                     // Локальные шрифты Golos Text
│   │   └── scss                      // SCSS-стили
│   │       ├── abstracts             // Токены и брейкпоинты
│   │       └── main.scss             // Глобальные стили
│   ├── components                    // Переиспользуемые Vue-компоненты
│   │   ├── catalog                   // Компоненты каталога товаров
│   │   └── layout                    // Компоненты лейаута
│   ├── composables                   // Переиспользуемая бизнес-логика
│   ├── constants                     // Константы приложения
│   ├── pages                         // Страницы (роуты) Nuxt
│   │   ├── catalog/index.vue         // Страница каталога
│   │   └── index.vue                 // Главная страница
│   └── types                         // TypeScript-типы
├── public                            // Публичные статические файлы
└──  server                            // Серверная часть Nuxt (Nitro)
    └── api/products.get.ts           // Прокси-эндпоинт получения товаров

```
